Core Features

Industrial-grade architecture designed for performance and flexibility.

High-Performance Script Hosting & Logic Decoupling

  • Dual-Language Collaboration: C++/Lua bidirectional binding via LuaBridge. The engine handles rendering and physics while Lua manages Actor lifecycle logic (OnStart, OnUpdate).
  • Dynamic Property Injection: Runtime overriding of Lua component member variables through JSON configuration — tweak game values without recompilation.
  • Exception Isolation: Script call exceptions are captured at the C++ layer using pcall, ensuring script-level errors never crash the engine process.

Scene Management & Serialization

  • Actor Templates & Inheritance: JSON-based Actor template system, similar to Unity Prefabs, with property overriding.
  • Global Persistence (DontDestroyOnLoad): Object persistence across scene transitions — global managers like audio controllers and score trackers maintain state between scenes.

Physics Middleware Integration

Deep integration with Box2D physics engine, exposing a full-featured component system.

Box2D-Powered Physics

  • Hybrid C++/Lua Components: Native Rigidbody and Collider C++ components with APIs exposed to the scripting layer, balancing high-frequency physics updates with developer-friendly interfaces.
  • Collision Filtering: Fine-grained collision filtering via categoryBits and maskBits, reducing unnecessary collision callbacks (ReportFixture) and optimizing CPU load in complex multi-body scenes.
  • Trigger System: Callback-based trigger mechanism enabling real-time monitoring of physics interaction events from the script layer.

Data-Oriented Design & Memory Optimization

Achieving massive particle throughput through cache-friendly architecture.

DOD Particle System

  • Cache-Friendly Layout: Replaced per-object storage with contiguous Structure-of-Arrays (SoA) memory layout, maximizing spatial locality and dramatically reducing CPU cache misses.
  • SIMD-Friendly Updates: Stripped non-computational class members from update loops, focusing solely on core computation and boosting instruction throughput for tens of thousands of active particles per frame.
  • Performance Leap: Several times improvement in particle processing capacity over traditional component-based updates on equivalent hardware.
